Protocols


IP

To determine an IP address, you can follow these steps:

Open a command prompt on your computer. Type "ipconfig" and press Enter. Look for the "IPv4 Address" line under the network adapter you're currently using. This will display the IP address of your computer on that network. Alternatively, you can visit a website that will display your public IP address, such as "whatismyip.com" or "ipchicken.com". These websites will show you the public IP address.

DNS

DNS records are used to translate domain names into IP addresses, allowing computers to locate servers on the internet. Here is a brief description of some common DNS record types:

A Record: This is the most common type of DNS record, and it maps a domain name to an IP address.

AAAA Record: Similar to A records, but used for IPv6 addresses.

MX Record: This record maps a domain name to the mail exchange servers that are responsible for handling email messages sent to that domain.

CNAME Record: This is an alias record that maps one domain name to another. It is often used for subdomains or to create a shorter, easier-to-remember URL.

TXT Record: This record can contain any arbitrary text and is often used for verification or authentication purposes.

SRV Record: This record is used to specify the location of a service on a domain, such as a specific port for a particular protocol.

NS Record: This record specifies the authoritative name servers for a domain.

These are just a basic of the many types of DNS records that exist. Each record type serves a specific purpose in the DNS system.

HTTP methods

HTTP methods are used to indicate the action to be performed on a resource identified by a URI. Here's a description of each HTTP method:

GET: Requests a representation of a resource. It retrieves information from the server without changing the state of the resource.

POST: Submits an entity to the specified resource, often causing a change in state or side effects on the server.

PUT: Replaces the entity of the specified resource with the new one provided in the request payload.

DELETE: Deletes the specified resource.

HEAD: Requests the header of a resource, without retrieving the actual representation of the resource.

OPTIONS: Requests information about the communication options available for the specified resource.

CONNECT: Converts the request connection to a transparent TCP/IP tunnel.

TRACE: Echoes back the received request, allowing clients to see what intermediate servers are changing or adding to the request.

PATCH: Partially modifies the specified resource by applying a set of changes specified in the request payload.

HTTP status codes

HTTP status codes are three-digit numbers returned by a server to indicate the status of a client's request. Here are some common HTTP status codes and their meanings:

200 OK: The request was successful, and the server has returned the requested data.
201 Created: The server has successfully created a new resource.
204 No Content: The server has fulfilled the request, but there is no new information to send back.

400 Bad Request: The server could not understand the request due to invalid syntax.
401 Unauthorized: The client does not have proper authentication credentials for the resource.
403 Forbidden: The client is authenticated, but does not have access to the requested resource.
404 Not Found: The server cannot find the requested resource.

500 Internal Server Error: The server encountered an error while processing the request.
503 Service Unavailable: The server is currently unavailable due to maintenance or overload.
